Architecture Repository

Reference Library

The reference library is where architecture becomes reusable. It should contain the small number of patterns, templates, and examples that teams can pull into real delivery work without major rewriting.

What belongs here

  • Reference architectures for common solution shapes
  • Starter templates for reviews, decisions, and transition planning
  • Example diagrams and document structures
  • Links to articles that explain when and how to use an asset

What good looks like

A useful reference asset is short, adaptable, and opinionated enough to save time. It should be understandable by project teams without needing a separate workshop to decode it.

Starter assets

  • Architecture artifact starter kit
  • Integration boundary guidance for ERP work
  • Capability map framing for planning conversations

Suggested library organization

Views

Current-state, target-state, and context diagrams.

Decisions

ADRs, exceptions, and review outcomes.

Patterns

Integration, data, security, and platform reference material.
