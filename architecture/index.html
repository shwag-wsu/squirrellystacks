Suggested way to use this repository

Start with the landscape

Use current-state and target-state views to frame the conversation and identify where change is really needed.

Pull from the reference library

Reuse templates and patterns instead of creating one-off artifacts for each initiative.

Capture decisions in the log

Keep enough rationale to explain why a path was chosen, what constraints mattered, and what to revisit later.
